# Product Context

## Why This Exists
Andrew Shindyapin needs a professional portfolio website to showcase projects built during Gauntlet AI Cohort 2 and present his work to potential employers and collaborators.

## Problems It Solves
1. **Showcasing Multiple Projects**: Provides elegant way to display diverse applications (web, mobile, desktop)
2. **Professional Presence**: Creates polished online presence for career opportunities
3. **Easy Contact**: Enables interested parties to reach out directly
4. **Project Documentation**: Preserves and presents project details with videos and descriptions

## How It Should Work

### User Journey
1. **Discovery**: Visitor lands on homepage, immediately sees project grid
2. **Exploration**: Clicks project card to view video demonstration in modal
3. **Learning More**: Reads full project description, tech stack, and accesses links
4. **Connection**: Navigates to About page to learn about Andrew
5. **Contact**: Uses contact form to reach out

### User Experience Goals
- **Minimal Friction**: No unnecessary clicks or loading delays
- **Visual Appeal**: Clean, modern design that doesn't distract from content
- **Information Clarity**: Easy to understand what each project does
- **Professional Tone**: Polished but approachable presentation

## Content Structure
- **Projects**: Video demonstrations with detailed descriptions and tech stacks
- **About**: Professional bio highlighting experience and interests
- **Contact**: Simple, functional form for inquiries

## Design Philosophy
- **Less is More**: Minimal design that highlights projects
- **Content First**: Projects take center stage, no unnecessary hero sections
- **Performance Focused**: Fast loading, optimized assets
- **Accessible**: Works for everyone, regardless of device or ability
